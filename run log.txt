1. FOUNDATION SERVICE FAILURE (CRITICAL – SYSTEM BREAKING)

ERROR:
- Flyway migration failure during startup
- Application fails to start completely

ACTUAL ERROR:
ERROR: value too long for type character varying(26)

FAILED QUERY:
INSERT INTO tenants (...)
VALUES ('01HTENANT000000000000000001', ...)

ROOT CAUSE:
- Column length mismatch in DB schema
- tenants.id is VARCHAR(26)
- Provided value length > 26 characters

WHY THIS IS HAPPENING:
- Hardcoded ID in migration script exceeds DB column size
- Schema and seed data are inconsistent

IMPACT:
- Foundation service does NOT start
- Dependent services may fail (tenant resolution, shared configs)
- System partially broken

FIX:
- Either:
  1. Increase column size (VARCHAR(36) or more)
  OR
  2. Reduce ID length in migration script
- Ensure all services use consistent ID format (UUID vs custom)

2. KAFKA TOPIC / LEADER NOT AVAILABLE (CRITICAL RUNTIME ISSUE)

ERROR:
LEADER_NOT_AVAILABLE for topics:
- email.bounced
- email.complaint

ROOT CAUSE:
- Topics not fully initialized when consumer starts
- Kafka broker not ready OR topic auto-creation delay
- No explicit topic creation before consumer subscription

IMPACT:
- Deliverability service cannot consume messages initially
- Event-driven flow broken temporarily
- Retry/rebalance cycles triggered

FIX:
- Pre-create topics before service startup
- OR add startup retry/backoff for Kafka consumers
- OR delay consumer startup until Kafka is ready
- Ensure proper broker readiness
