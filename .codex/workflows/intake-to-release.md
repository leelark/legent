# Intake To Release Workflow

1. Capture user request and constraints.
2. Requirements Analyst writes acceptance criteria and non-goals.
3. Product Manager maps user value, modes, and priority.
4. CTO and Principal Architect identify service boundaries and risk.
5. Program Manager decomposes into disjoint work items.
6. Assign up to 6 independent agents when delegation is authorized.
7. Implement smallest coherent slices.
8. Test Architect selects validation profile.
9. QA verifies behavior and regressions.
10. Security/Performance/DevOps review if their surfaces are touched.
11. Release Manager decides release posture.
12. Documentation Engineer updates docs and memory.

No slice exits release review without validation evidence or explicit residual risk.
