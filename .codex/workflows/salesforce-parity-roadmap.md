# Salesforce Parity Roadmap Workflow

Use official current sources for market capability checks when making parity claims.

## Capability Areas

- Email Studio: template creation, reusable blocks, dynamic content, personalization, testing, approvals, test sends.
- Contact Builder: subscribers, data extensions, data relationships, imports, preferences, data views.
- Journey Builder: event triggers, waits, decisions, goals, exits, re-entry, versioning, simulation, monitoring.
- Automation Studio: imports, SQL/query activities, extracts, file/object storage, schedules, send automation.
- Deliverability: authentication, warmup, suppressions, feedback loops, DMARC, reputation, safety checks.
- Analytics: campaign, journey, audience, deliverability, provider, and engagement analytics.
- AI: content assistance, predictive segments, send-time optimization, frequency optimization, channel decisioning, anomaly detection.
- Enterprise: governance, approvals, environments, audit, RBAC, SSO/SCIM, evidence-based release.

## Output

Each parity pass records:

- Current Legent capability.
- Baseline market capability.
- Gap.
- Risk.
- Required implementation.
- Required validation.
- Release posture.
