# Security Findings

Findings:

Source: `infrastructure/kubernetes/overlays/production/delete-base-egress-policy.yml`, `network-policy.yml`, `validate-production-overlay.ps1`. Validation: production overlay validator, Kustomize render, and full release gate passed. Residual: live external provider/managed-service egress CIDRs or approved CNI FQDN policy are still required before promotion.

Source: `frontend/src/lib/api-client.ts`, `frontend/tests/e2e/api-client-context.spec.ts`. Validation: focused Playwright API-context tests passed; frontend lint passed with existing `no-explicit-any` warnings.
- 2026-05-16, resolved: SCIM token copy failure toasts no longer include the raw token value. Source: `frontend/src/components/admin/FederationConfigPanel.tsx`. Validation: frontend lint passed.

Source: `shared/legent-security/src/main/java/com/legent/security/UnsafeMethodOriginGuardFilter.java`. Validation: focused shared-security Maven tests passed.

- 2026-05-16, resolved: platform webhook create validation now rejects unsupported, malformed, and excessive subscribed event names, trims `secretKey`, and requires a minimum 32-character mixed-character secret before persistence. Source: `services/platform-service/src/main/java/com/legent/platform/controller/WebhookController.java`. Validation: focused platform webhook Maven tests passed.
- 2026-05-16, resolved: outbound webhook dispatch/retry now uses a dedicated webhook `WebClient` wrapper with connector-time resolved-address validation, so private/reserved DNS answers returned during the actual connect path are rejected instead of relying only on pre-dispatch validation. Production profile direct webhook egress is disabled unless `legent.webhooks.outbound.proxy-url` is configured, preserving a fail-closed proxy path where full address pinning would risk TLS/SNI behavior. Source: `OutboundUrlGuard`, `PublicAddressValidatingAddressResolverGroup`, `WebClientConfig`, `WebhookDispatcherService`, `WebhookRetryService`. Validation: focused shared-common/platform Maven webhook tests passed.
- 2026-05-16, resolved: automation workflow trigger idempotency no longer marks events processed before workflow start succeeds; pending claims are marked complete only after start, released on failure, and publish failures are rethrown. Validation: affected backend module test sweep passed.
- 2026-05-16, resolved: deliverability feedback-loop idempotency no longer marks bounce/complaint events complete before suppression/reputation side effects finish; pending claims are completed after side effects or released on failure. Validation: affected backend module test sweep passed.
- 2026-05-16, resolved: audience subscriber intelligence no longer catches/logs parse or repository failures after idempotency registration; failures propagate so the transaction can roll back and Kafka retry can engage. Validation: affected backend module test sweep passed.
