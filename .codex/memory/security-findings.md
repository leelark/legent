# Security Findings

Last updated: 2026-05-16.

Findings:

- 2026-05-16, resolved: foundation tenant provisioning no longer logs and acknowledges unexpected signup/bootstrap failures; critical bootstrap requested/completed publishes are awaited and rethrown on failure. Validation: affected backend module test sweep passed.
- 2026-05-16, resolved: automation workflow trigger idempotency no longer marks events processed before workflow start succeeds; pending claims are marked complete only after start, released on failure, and publish failures are rethrown. Validation: affected backend module test sweep passed.
- 2026-05-16, resolved: deliverability feedback-loop idempotency no longer marks bounce/complaint events complete before suppression/reputation side effects finish; pending claims are completed after side effects or released on failure. Validation: affected backend module test sweep passed.
- 2026-05-16, resolved: audience subscriber intelligence no longer catches/logs parse or repository failures after idempotency registration; failures propagate so the transaction can roll back and Kafka retry can engage. Validation: affected backend module test sweep passed.

- 2026-05-13, resolved: wildcard Kafka trusted package entries in tracking, platform, campaign, automation, deliverability, and delivery service YAML files were narrowed to `java.lang,java.util,com.legent.kafka.model`.
- 2026-05-13, resolved: service `application.yml` and Kubernetes base defaults now use `SPRING_JPA_HIBERNATE_DDL_AUTO:validate`; intentional test `create-drop` remains. Impact: absent env/overlay fails closed instead of mutating schemas. Residual risk: startup may now reveal incomplete Flyway migrations.
- 2026-05-13, resolved: shared `TenantFilter` now rejects `X-Workspace-Id` and `X-Environment-Id` conflicts against JWT-populated `TenantContext` and preserves authenticated workspace/environment when headers are omitted. Source: `shared/legent-security/src/main/java/com/legent/security/TenantFilter.java`. Validation: `.\mvnw.cmd -pl shared/legent-security -am test`.
- 2026-05-13, resolved: service Kafka consumers in delivery, tracking, platform, deliverability, automation, campaign, and audience now rethrow unexpected processing failures after logging, and shared Kafka retry exhaustion publishes to declared central `kafka.dead-letter`. Validation: impacted Maven slices passed.
- 2026-05-13, source `AGENTS.md`: inbox placement must not be guaranteed. Impact: product/security/compliance risk if UI/API claims impossible guarantee. Next action: audit copy before release.
- 2026-05-13, resolved: tracking `/ws/analytics` now requires authentication and tenant/workspace handshake context; refreshed access tokens preserve workspace/environment claims so browser WebSocket clients do not need custom headers. Validation: focused Maven slice passed.
- 2026-05-14, resolved: service and Kubernetes base Flyway defaults now use `baseline-on-migrate=false`, `validate-on-migrate=true`, and `out-of-order=false`; DDL defaults remain `validate`. Impact: absent env/overlay fails closed. Residual risk: Kubernetes base still includes local/nonprod dependencies such as MailHog/local URLs, so production must use the production overlay and release validators.
- 2026-05-13, resolved: raw HTML frontend sinks now use shared DOMPurify sanitizer profiles, and content-service sanitization uses jsoup DOM parsing/policy tests for script, event handler, encoded URL, form, srcset, SVG/MathML, and unsafe CSS vectors. Residual risk: keep edit/export raw HTML contract documented and do not render raw authoring HTML without sanitizer.
- 2026-05-13, resolved: identity `switchContext` and delegation token exchange now require active account/tenant/workspace membership before minting JWT workspace context, including delegated subject membership checks; legacy no-account fallback cannot mint arbitrary workspace IDs. Residual risk: tenant-only content/platform/data-extension schemas still need workspace scoping decisions below.
- 2026-05-13, open: many service workspace filters check header/JWT scope presence, not membership/authorization. Impact: service-level workspace authorization remains incomplete outside identity token minting. Next action: add membership/authorization checks against identity/foundation workspace membership where services accept workspace-scoped requests.
- 2026-05-13, open: content and platform schemas are tenant-only. Audience data extensions were fixed on 2026-05-13 with workspace columns/service filters/tests. Impact: remaining content/platform surfaces can expose data across workspaces within a tenant unless product intentionally scopes these globally. Next action: decide ownership scope, add workspace columns/filters or document tenant-global behavior.
- 2026-05-13, resolved: shared `KafkaConsumerConfig` no longer hardcodes `com.legent.*`; it defaults to `java.lang,java.util,com.legent.kafka.model`, `EventEnvelope`, and disabled type headers.
