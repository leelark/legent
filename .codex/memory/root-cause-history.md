# Root Cause History

Last updated: 2026-05-14.

- 2026-05-14: Fixed root cause for delivery/tracking cross-scope event writes: consumers mixed envelope scope with payload tenant/workspace fields and idempotency claims could be recorded under one scope while side effects executed under another. Change rejects envelope/payload mismatches, normalizes payload scope to envelope scope before side effects, and separates idempotency claim/complete/release with nullable pending-claim migrations.
- 2026-05-14: Fixed root cause for webhook secret disclosure: platform webhook controllers returned the JPA entity directly and frontend API types accepted `secretKey` in responses. Change makes `secretKey` JSON write-only and returns `secretConfigured` for UI state.
- 2026-05-14: Reduced root cause for platform/content authorization gaps: platform config and content landing-page publishing depended mostly on authenticated tenant context, not explicit RBAC. Change adds config/platform and content/template permission gates. Residual: the foundation settings bridge still needs an explicit service-token or caller-token relay design.
- 2026-05-14: Fixed root cause for release gates masking failed native tools on Windows PowerShell: `$ErrorActionPreference=Stop` did not make `mvnw.cmd`, `npm`, `docker`, or `kubectl` failures terminating. Change adds a native command wrapper that checks `$LASTEXITCODE`.
- 2026-05-14: Fixed root cause for unsafe Flyway defaults in service configs: several services relied on environment/overlay overrides while local defaults allowed baseline or out-of-order migration behavior. Change aligns all service-local defaults to baseline false, validation true, out-of-order false.
- 2026-05-14: Reduced root cause for public landing-page form abuse: stripping only submit target attributes left real form semantics in rendered public HTML. Change unwraps/removes `<form>` tags in frontend and backend sanitizers and keeps controls inert until a controlled submission endpoint exists.
- 2026-05-14: Fixed root cause for unsafe legacy data-extension workspace assumption: V16 could only assign historical tenant-scoped data extensions to `workspace-default` because the audience schema had no authoritative workspace mapping. V17 now fails closed when legacy rows exist unless operators provide reviewed mapping metadata, validates that mapping before applying it, and leaves clean installs unaffected.
- 2026-05-13: Fixed root cause for content sanitizer bypass risk: `EmailContentValidationService` removed tags and attributes with regexes, which could miss parser-normalized HTML such as encoded JavaScript URL schemes, dangerous form submit targets, `srcset`, foreign-content SVG/MathML, and unsafe CSS URL variants. Change uses jsoup parser output plus an explicit allowlist policy for tags, attributes, URL schemes, and inline CSS rules.
- 2026-05-13: Fixed root cause for unsafe DDL default: service configs and Kubernetes base used local/convenience `ddl-auto:update` defaults, relying on Compose/production overlays to override. Change set default to `validate` so absent env/overlay fails closed.
- 2026-05-13: Fixed root cause for Compose health drift: validator hardcoded frontend port 3003 while Compose default maps host port 3000; Redis healthcheck ignored optional Redis auth. Change made validator read `FRONTEND_HOST_PORT` and Redis healthcheck authenticate only when password exists.
- 2026-05-13: Fixed root cause for broad Kafka deserialization trust: six service YAML files used `spring.json.trusted.packages:*`, and shared `KafkaConsumerConfig` hardcoded `com.legent.*`. Change narrowed both to current envelope-only needs.
- 2026-05-13: Fixed root cause for JWT workspace context overwrite: `TenantFilter` only conflict-checked tenant ID and unconditionally wrote workspace/environment from request headers, allowing authenticated context to be cleared or replaced. Change made JWT-populated tenant/workspace/environment authoritative when present.
- 2026-05-13: Reduced root cause for resolved-audience oversized Kafka events: `AudienceResolutionConsumer` previously converted the whole filtered audience into one subscriber-map list and published a single `isLastChunk=true` event. Change publishes bounded deterministic chunks; full in-memory resolution remains a separate root cause.
- 2026-05-13: Reduced root cause for full-audience materialization and suppression fail-open behavior: `AudienceResolutionConsumer` still resolves LIST/SEGMENT IDs up front, but default all-subscriber resolution no longer loads every subscriber ID or every Subscriber entity before suppression; it uses keyset candidate pages, page-local suppression checks, and one-pass chunk lookahead for correct final chunk metadata. `DeliverabilityServiceClient` no longer returns an empty suppression set on failed or malformed responses, and the listener rethrows suppression/publish failures for retry/DLQ handling.
- 2026-05-13: Fixed root cause for shared Kafka tenant hot-partitioning default: generic `EventPublisher.publish(topic, envelope)` keyed by tenant ID for all topics. Change forces high-volume topics onto non-tenant routing keys while preserving low-volume tenant fallback.
- 2026-05-13: Reduced root cause for delivery send-request silent acknowledgements: `DeliveryEventConsumer` caught all exceptions and returned after logging. Change rethrows after logging so listener retry/DLQ policy can engage.
- 2026-05-13: Fixed root cause for Kafka listener silent acknowledgements and inert DLQ path: shared `DefaultErrorHandler` had no recoverer, and automation/tracking/deliverability/campaign/audience/platform listeners swallowed broad processing exceptions. Change wires `DeadLetterPublishingRecoverer` to declared central `kafka.dead-letter` and rethrows unexpected processing failures so retry and DLQ engage; explicit invalid/malformed-event drops still return.
- 2026-05-13: Fixed root cause for authenticated analytics WebSocket breakage after token refresh: refresh minted access tokens without workspace/environment claims while the WebSocket handshake required workspace context browsers could not provide as a custom header. Change preserves workspace/environment from the prior signed access token during refresh and lets the handshake use authenticated principal workspace.
- 2026-05-13: Fixed root cause for production overlay drift: production reused base ingress hosts and base `legent/*:latest` image tags without a render-time drift gate, and ClickHouse DB stayed at the base `default` value. Change pins production images, uses production ingress hosts, sets `CLICKHOUSE_DB=legent_analytics`, and adds a production overlay validation step to release gate.
- 2026-05-13: Fixed root cause for final Compose runtime validation failure: the health validator trusted a stale/default frontend host port while Compose override published frontend on 3003, and the new `/ws/analytics` Nginx location repeated `proxy_http_version` after including common proxy params. Change resolves frontend host port from live Compose metadata and removes the duplicate directive.
- 2026-05-13: Fixed root cause for LIST/SEGMENT audience heap risk: resolution built Java include/exclude subscriber ID sets before paging subscribers. Change parses audience refs and delegates include/exclude membership logic to DB-backed keyset candidate queries.
- 2026-05-13: Fixed root cause for arbitrary workspace JWT context minting: identity context switching validated tenant membership but reused caller-supplied workspace ID without membership proof. Change requires active workspace membership for context switch and delegated token exchange.
- 2026-05-13: Fixed root cause for route/controller drift: validation only compared route-map to Nginx/ingress, so active controller roots could be unreachable while the validator passed. Change adds missing roots, removes unused `/events`, and validates controller roots against the route map.
- 2026-05-13: Fixed root cause for production ingress route shadowing: validator scanned ingress paths in YAML order, but ingress-nginx renders regex paths by descending path length, allowing the broad foundation `/api/v1/(...|admin|public|...)` regex to win over service-specific `/api/v1/public/landing-pages` and `/api/v1/admin/(webhooks|search)` paths. Change splits admin/public into shorter foundation paths and validates rendered precedence plus critical expected services.
- 2026-05-13: Fixed root cause for delegated workspace token spoofing: delegation verified the caller context but not the delegated subject's active membership in the target tenant/workspace. Change validates both users and target membership before minting delegated JWTs.
- 2026-05-13: Fixed root cause for CSS sanitizer escape bypass: URL scheme checks normalized HTML entities but not CSS escape sequences before scanning `url(...)`. Change decodes CSS escapes before compact scheme checks.
- 2026-05-13: Fixed root cause for landing-page arbitrary form submission: form `action` and button `formaction` were treated like general URL attributes, allowing external HTTP(S) endpoints from rendered public landing pages. Initial change stripped both attributes; 2026-05-14 change now unwraps/removes `<form>` tags entirely until a Legent-controlled submission endpoint policy is defined.
- 2026-05-13: Fixed root cause for audience data-extension cross-workspace visibility: data extension owner rows and records were tenant-only while the audience service required workspace request context. Change adds workspace columns, scoped indexes, tenant+workspace repository methods, workspace assignment on create/addRecord, and same-tenant cross-workspace tests.
- 2026-05-13: Reduced root cause for campaign batch JSON retention: completed send batches kept recipient payload JSON after successful processing. Change clears payload only on completed batches while preserving partial/retry payloads.
- 2026-05-13: Fixed root cause for production release gate masking broad egress: validation checked image/local/ClickHouse drift but not rendered NetworkPolicy egress, while production inherited broad base `allow-legent-egress`. Change makes production overlay validation fail closed on inherited broad egress until a reviewed allowlist can be authored.
- 2026-05-13: Fixed root cause for Loop 4 infra drift: production default-deny ingress lacked an explicit frontend allow policy, tracking routes inherited the global ingress 100r/s annotation, route-map order allowed parent prefixes before service-specific children, and Compose gateway API health validation still probed `/api/health`. Change adds frontend ingress allow, splits tracking ingress without the shared rate annotation, rejects parent-before-child route-map ordering, and probes `/api/v1/health`.
- 2026-05-13: Fixed root cause for foundation public-content cross-workspace mutation: admin update/publish paths accepted a record ID and used bare `findById`, bypassing the tenant/workspace filters used by list and public reads. Change adds an `id + tenant_id + workspace_id` repository method, uses it for update/publish, and adds denial tests for cross-workspace IDs. Related foundation surfaces had workspace-aware DTOs but allowed conflicting explicit workspace IDs; workspace resolvers now reject conflicts against `TenantContext`.
- 2026-05-13: Fixed root cause for campaign handoff failure loss: batching/send execution caught unexpected processing and async publish failures, marked work as progressed, and could advance counters before delivery feedback existed. Change rethrows unexpected failures for retry/DLQ, joins durable publish handoff, preserves failed payloads for retry, and leaves recipient outcome counters to delivery feedback.
- 2026-05-13: Fixed root cause for delegated token requester spoofing: delegation checked the caller context and delegated subject but did not prove the requesting user/account had active membership in the target workspace. Change validates requester account/user state and requester membership before delegated-subject checks.
- 2026-05-13: Fixed root cause for workspace selector persistence drift: frontend header hydration matched workspaces by tenant only and wrote localStorage during list reconciliation, allowing one workspace choice to overwrite another workspace under the same tenant. Change matches tenant plus workspace and only persists after successful context switch.
- 2026-05-13: Fixed root cause for audience resolution duplicate suppression/loss: idempotency was recorded before all side effects and async Kafka publish success were known. Change moves the listener into a transaction and waits for publish futures so DB/publish failures roll back or retry visibly.
- 2026-05-13: Fixed root cause for campaign event loss after partial side effects: idempotency completion was conflated with initial claim, and critical publish futures could fail after local state moved forward. Change separates claim/complete/release, waits for durable publish futures, requeues unfinished batches, and reconciles terminal batch failure.
- 2026-05-13: Fixed root cause for production runtime secret drift: the production ExternalSecret did not render all environment keys required by service configs, and validation did not check those keys. Change adds missing datasource, tracking, and delivery cryptographic keys and validates rendered `secretKey` entries.
- 2026-05-13: Fixed root cause for public landing pages failing closed to users with stale cookies: the shared JWT filter rejected invalid auth cookies before public `permitAll` routing could apply, and the frontend reused the credentialed API client for public reads. Change makes only documented public paths optional-auth tolerant and uses a no-credential public client.
- 2026-05-13: Fixed root cause for forged environment context during workspace switch: identity validated tenant/workspace membership but embedded caller-supplied environment ID into the JWT. Change drops client-supplied environment context until authoritative validation exists, and frontend clears stale local environment state when absent from server context.
- 2026-05-13: Fixed root cause for SSO/SCIM tenant-header deadlock: identity security made those endpoints public, but shared `TenantFilter` still required `X-Tenant-Id` before the controller/service could run. Change exempts those public roots from tenant headers while preserving service-level SCIM bearer validation.
- 2026-05-13: Fixed root cause for foundation dynamic cross-workspace mutation: several helper flows loaded or updated records by `id + tenant_id` despite workspace-scoped request context. Change adds workspace-scoped ID read/update paths and uses them in global, differentiation, and performance helper flows.
- 2026-05-13: Fixed root cause for `.env.example` validation drift: placeholder values were intentionally non-secret but shorter than the minimum lengths enforced by `validate-env.ps1`. Change lengthens placeholders so example validation can pass without real secrets.
- Known risk pattern: non-delivery catch/log consumers and in-memory high-volume audience/campaign paths remain unsafe for production scale unless redesigned.
