# intake

Purpose: convert a request into actionable work.

Steps:

1. Restate objective.
2. Identify user-visible outcome and non-goals.
3. Identify service/module owners.
4. Identify security, tenant, performance, schema, route, and release impacts.
5. Write acceptance criteria.
6. Create or update `.codex/memory/active-work-items.md`.
7. Choose validation profile from `.codex/workflows/validation-gates.md`.
