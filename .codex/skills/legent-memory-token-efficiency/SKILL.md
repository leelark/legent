# Legent Memory Token Efficiency

1. Read `.codex/workflows/token-efficient-memory.md`.
2. Keep memory files concise: durable facts, risks, decisions, fixes, and next actions only.
3. Put detailed activity in `.codex/audit/events/YYYY-MM-DD.jsonl`.
4. Put recovery details in checkpoints.
5. Put narrative evidence in reports.
6. Keep active-work memory limited to live state and next handoff.

Never store secrets, raw logs with credentials, customer data, or bulky command output.
