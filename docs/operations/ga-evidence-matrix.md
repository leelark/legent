## Evidence Storage

Store target-environment transcripts or signed summaries under `docs/operations/` or `.codex/reports/` with timestamp, environment, release version, commands run, pass/fail result, and owner. Keep secrets, raw tokens, credentials, private keys, and sensitive customer data out of evidence artifacts.

For GA promotion, store a dedicated evidence directory such as `docs/operations/evidence/2026-05-16-rc1/` with `ga-evidence-manifest.json`. The validator accepts either an `evidence` object keyed by artifact ID or an `artifacts` array with `type`, `status`, `date`, and `files` fields. Every referenced file must be a local file inside the evidence directory.

Required artifact IDs are: `synthetic-smoke`, `live-load`, `restore-drill`, `ci-gitleaks`, `ci-trivy`, `ci-sbom`, `monitoring-alert-routing`, `tls-cert-ownership`, `restricted-admission`, `registry-digest`, `registry-sbom`, `registry-signature`, and `registry-provenance`.
